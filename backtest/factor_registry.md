# 因子登记册 (Factor Registry)

把散落在会话记忆里的因子裁决 (假设 / 方向 / 前瞻期限 / 通过门槛 / 已试次数 / 淘汰标准)
固化进仓库, 让"只读 repo 的未来的你或子 agent"也能看到**为什么某因子在评分里、某候选被否**,
避免重复实验或悄悄回灌已否决的因子。

这是**登记册模板 + 存量登记**, 不是统计引擎。新增/淘汰因子时同步更新本表 (与
`btc_web/btc_dashboard/scoring.py` 桶配置、`backtest/factors.py` 打分口径互为对照)。

> 数据源与打分口径以 `scoring.py` (现网) / `backtest/factors.py` (回测) 为准; 本表只记
> **假设与裁决**, 不重复阈值 (阈值一致性由 `tests/test_consistency.py` 守卫)。

## 准入门槛 (预设, 施测前先写下)

| 门槛 | 内容 |
|---|---|
| 机械门 (必要非充分) | 战术因子: 分位分 vs 前瞻收益 `|IC| ≥ 0.08` 且三窗 (7/14/30d 或 90/180/365d) 符号一致 |
| 稳健 | 跨 regime 稳 (full / post-2017 / post-2021 三窗不翻号); 换 RV/窗口参数不崩 |
| 正交增量 | 与现有同轴因子 (MVRV-Z / 幂律 / Mayer 分位…) `|corr|` 不过高 (经验 < ~0.8); 入桶模拟须真提升 Sharpe/IC |
| 经济可解释 | 有 P 测度、跨多年周期的机制, 不是 Q 测度短均值回归量误当 regime 变量 |
| 参照基线 | 最强单因子 = **幂律走廊** IC `+0.58 / +0.69 / +0.77` (full/17/21 @365d); 新周期因子须显著优于它才值得进桶 |

**机械门必要非充分**: `|IC|≥0.08` 隐含近 IID 假设, 而 BTC 高波动段聚簇于宏观周期, 有效自由度
常仅 order-10~50, 名义 n 是幻觉。过机械门仍须过稳健 + 增量 + 可解释三关 (VRP 判例: 机械过关却三者皆缺 → display_only)。

## 裁决方法论前科 (踩过的坑, 复用判例)

1. **全样本指标会说谎** → 按**现行体制 (2024+)** 裁决。交易所余额全样本 IC365 0.603 领先, 但优势全在前 ETF 时代; 2024+ 移除后 IC365 +0.085→+0.229 / Sharpe 0.89→1.20 → 移除。
2. **聚合窗改善 + 最新切片反指** → 按**最新切片**裁决。LTH 占比聚合窗 IC 好, 但 2025-01 起翻转为 −0.499 反指。
3. **因子裁决前必先刷新价格缓存**。实测缓存滞后 54 天可把 post-2024-10 IC 从真实 +0.069 美化到 +0.605。
4. **单 regime 伪回归**: 慢单调序列 × 单边行情 = 高 IC 伪影 (Liveliness 4y IC +0.79 / M2-108天 / RHODL 单牛腿, 均不可信)。
5. **多重检验**: 不用 Bonferroni / deflated-Sharpe (假设可数的独立试验, 现实不干净存在); 以现行体制分层 + 冗余相关 + 最新切片裁决替代。见文末试验计数注记。

## 试验计数注记 (多重检验成本)

截至 2026-07, 已系统筛查 **~20+ 候选** (周期 6 + 币龄 7 + 期权/预测市场/VRP/HAR-RV/纯 RV 等),
全部留有 IC + 冗余 + regime 分层证据。**试验越多, 单次"过机械门"越廉价** —— 故:

- 不建正式试验计数器 / Bonferroni (需可数独立试验, 现实不干净); 以现行体制分层 + 冗余相关 +
  最新切片裁决作为更贴领域的防线。
- 新候选须显著优于最强单因子基线 (幂律 IC +0.77@365 post21), 而非仅"IC 为正"。
- 现行头名基线 (样本内, 2026-07-18 修正 MVRV-Z σ 口径后): 全样本 IC365 **0.506**,
  滞回 Sharpe **1.01**, 最大回撤 **−44.6%** (旧数 0.514/1.00 含 14.3% 天数口径漂移, 勿引用)。
