# 🧭 BTC Compass · 周期与时机仪表盘

> 本项目是 [BTC-Dashboard](https://github.com/jackyhe1819-create/BTC-Dashboard) 的实验性分叉。
> 同一套数据底盘，不同的评分哲学 —— 用于 A/B 对比两种评分体系的实际表现。

## 与原版的核心区别

### 1. 双评分输出（原版：单一总分）

| 评分 | 回答的问题 | 变化节奏 | 因子构成 |
|---|---|---|---|
| **🧭 周期仓位分** | 该配多少仓位 | 以周为单位 | 估值 + 筹码 + 资金流 + 趋势确认 |
| **⚡ 短期战术分** | 现在是不是好的操作时点 | 以天为单位 | 杠杆 + 动量 + 情绪 |

原版把短期指标压到 8% 权重揉进单一总分 —— 短期信号被淹没，长期信号又被噪音污染。
两个问题本就该由两个分数回答。

### 2. 因子分桶去相关（原版：8 个均线变体合占 65%）

原版的 Mayer、2Y MA、200W、Golden Ratio、Pi Cycle、Ahr999、幂律、均衡价格本质都是
"价格相对均线/时间模型的伸展度"，同一因子被数了八遍。本版按信息来源分桶：

```
周期分 = 趋势伸展 25% + 链上筹码 25% + 资金流 20% + 趋势确认 15% + 矿工经济 10% + 时间周期 5%
战术分 = 杠杆温度 40% + 动量结构 35%(含链上 SOPR) + 市场情绪 25%
```

### 3. 滚动分位数归一化（原版：绝对阈值）

原版的逃顶阈值（2Y MA ×5、Mayer 2.4、200W +300%）按 2013/2017 周期振幅校准，
随着周期振幅递减，这些阈值在真实顶部永远打不出 -1。
本版对估值类指标改用 **当前值在过去 4 年的分位数**，自适应振幅衰减。

### 4. 修复原版计分缺陷

- **ETF 评分用真实净流入**（SoSoValue 日度申赎数据），对称打分 —— 原版用成交量且永不给负分
- **失败指标剔除权重** —— 原版按 0 分中性票计入，拖累总分
- **恐惧贪婪仅极值计分**（≤30 / ≥70）—— 原版在牛市常态区间 55-75 系统性给 -0.5
- **资金费率用 7 日均值** —— 原版用单次 8 小时快照，噪音大

### 5. 新增九个正交因子

| 指标 | 数据源（免费） | 作用 |
|---|---|---|
| **MVRV-Z** | bitcoin-data.com | 链上周期估值黄金标准 |
| **STH 成本线** | bitcoin-data.com | 短期持有者成本 = 牛熊分界中线 |
| **NUPL** | bitcoin-data.com | 全网未实现盈亏的周期情绪分区 |
| **SOPR** | bitcoin-data.com | 链上割肉/止盈探测器（战术分） |
| **Puell Multiple** | bitcoin-data.com | 矿工收入周期（覆盖"关机币价"视角） |
| **Hash Ribbons** | mempool.space（本地计算） | 矿工投降→恢复的经典买点信号 |
| **稳定币增速** | DefiLlama | 场内弹药 / 流动性代理 |
| **期货年化基差** | OKX 季度合约 | 比资金费率更稳的杠杆温度计 |
| **趋势过滤器** | 本地计算（20W EMA + 200DMA 斜率） | 区分"便宜且企稳"与"接飞刀" |

链上慢变量统一 6 小时进程内缓存，不随 5 分钟仪表盘刷新重复请求免费 API。

## 运行

```bash
pip install -r btc_web/requirements.txt
cd btc_web && python3 app.py   # http://localhost:5050
```

## 部署（Render）

仓库根目录含 `render.yaml`（Blueprint），在 Render 控制台
**New → Blueprint → 选择本仓库** 即可一键部署。

│   ├── indicators_v2.py        # ★ 新增正交因子
│   ├── indicators_long.py      # 长周期指标

│   ├── indicators_aux.py       # 情绪/资金流指标
